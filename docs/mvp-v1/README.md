# MVP v1 Docs

This directory keeps the legacy MVP v1 document set as reference material.

## Structure

- `features/`: feature specs, architecture notes, issue writeups, and exploration docs
- `process/`: legacy mirrors of workflow rules that now live primarily under `docs/agent-workflow/`
- `planning/`: v1 follow-up task notes
- `product/`: product and UI notes
- `data/`: supporting reference data

## Use Rule

- Preserve these documents as v1 history unless they are explicitly being migrated or reduced to legacy mirrors.
- New v2 planning should be written under `docs/mvp-v2/`.
- When a v1 feature directory becomes dense, prefer splitting by document role before adding more top-level files.
