[feature] OpenCV 기반 프레임 추출 기능 개발

브랜치명: `feature/10-opencv-frame-extraction`

## 작업 목적
비디오 기반 자세 분석 또는 후속 영상 처리 기능을 구현하려면 원본 영상에서 원하는 기준으로 프레임을 안정적으로 추출할 수 있어야 한다. 이 작업은 OpenCV를 이용해 프레임 추출 기능의 기본 파이프라인을 만들고, 이후 스켈레톤 추출이나 분석 로직에서 재사용 가능한 입력 처리 기반을 마련하는 것을 목적으로 한다.

## 작업 내용
- OpenCV로 비디오 파일을 열고 프레임을 순차적으로 읽는 기능을 구현한다.
- 전체 프레임 추출과 간격 기반 샘플링 추출 방식을 지원한다.
- 추출된 프레임을 지정 경로에 이미지 파일로 저장하거나 후속 처리 모듈에 전달할 수 있도록 구조를 설계한다.
- 프레임 번호, 시간 정보, 저장 결과를 확인할 수 있는 메타데이터를 함께 정리한다.
- 잘못된 입력 파일, 읽기 실패, 저장 실패 상황에 대한 예외 처리를 추가한다.

## 완료 조건
- 비디오 파일을 입력받아 프레임을 정상적으로 추출할 수 있다.
- 추출 간격 또는 FPS 기준 옵션에 따라 결과 프레임 수가 의도대로 달라진다.
- 추출된 프레임이 지정한 경로에 저장되거나 재사용 가능한 데이터 형태로 반환된다.
- 비정상 입력에 대해 원인을 확인할 수 있는 예외 메시지가 제공된다.

## 참고 자료
- OpenCV `VideoCapture` 기반 비디오 프레임 읽기 방식
- 기존 비디오 처리 및 스켈레톤 추출 기능과의 연계 가능성 검토
- 향후 포즈 추출 파이프라인에서 사용할 프레임 전처리 요구사항

## 작업 내용
- OpenCV 기반으로 비디오 파일을 읽고 프레임을 순차적으로 추출하는 기능을 구현한다.
- 전체 프레임 추출 또는 지정 간격 샘플링 방식으로 프레임을 저장할 수 있도록 처리한다.
- 추출된 프레임을 이미지 파일 또는 후속 분석 파이프라인에서 사용할 수 있는 형태로 정리한다.
- 입력 파일 유효성 검사와 예외 처리 흐름을 추가해 실패 상황에서도 원인을 확인할 수 있도록 한다.

## 변경 사항 상세
- OpenCV `VideoCapture`를 이용해 비디오 메타데이터를 읽고 프레임 반복 처리 구조를 추가한다.
- 프레임 번호, 타임스탬프, 저장 경로 등 추출 결과를 추적할 수 있는 메타데이터 구조를 정의한다.
- FPS 기준 또는 N프레임 간격 기준으로 추출 옵션을 분리해 재사용 가능하게 설계한다.
- 잘못된 파일 경로, 지원되지 않는 포맷, 읽기 실패 상황에 대한 예외 메시지를 정리한다.

## 테스트 방법
1. OpenCV 실행 환경에서 프레임 추출 모듈을 실행한다.
2. 샘플 비디오 파일을 입력해 지정한 경로에 프레임 이미지가 생성되는지 확인한다.
3. 추출 간격 옵션을 변경해 저장되는 프레임 수가 예상과 일치하는지 확인한다.
4. 존재하지 않는 파일 또는 손상된 비디오 입력 시 예외 처리가 정상 동작하는지 확인한다.

## 스크린샷 / 결과

- 프레임 번호 및 메타데이터 로그 출력 확인
