# MPV v2 Issue Tracking

Use this structure to draft and maintain GitHub issues for `mpv v2`.

- `umbrella/`: top-level issue docs for scope, goals, risks, and linked work
- `sub-issues/`: one document per concrete task or chore

## Recommended Flow

1. Start from `umbrella/mvp-v2-umbrella.md`.
2. Add focused work items under `sub-issues/`.
3. Include `Parent: #<issue-number>` in each sub-issue draft.
4. Keep shared scope changes and cross-cutting decisions in the umbrella doc.
