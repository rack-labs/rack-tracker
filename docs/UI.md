## 랜딩페이지 UI 문서

이 문서는 `Stitch AI`에 전달할 랜딩페이지 UI 기획용 초안이다.

### 목적

- 산출물은 `MVP v1`을 소개하는 랜딩페이지다.
- 이 페이지는 제품의 핵심 문제의식, 동작 방식, 그리고 MVP 범위를 직관적으로 설명해야 한다.
- 최종 목표는 정식 플랫폼 전체가 아니라, 초기 핵심 기능 검증용 프로토타입을 명확하게 보여주는 것이다.

### 제품 개요

`MVP v1`은 사용자가 업로드한 운동 영상을 분석해 운동역학 기반 피드백을 제공하는 웹 서비스다.

동작 흐름은 다음과 같다.

1. 사용자가 자신의 운동 수행 영상을 업로드한다.
2. 시스템이 영상을 프레임 단위로 분리한다.
3. `MediaPipe`를 이용해 각 프레임에서 인체 스켈레톤 데이터를 추출한다.
4. 추출된 데이터를 로컬 백엔드 서버로 전달한다.
5. 서버에서 운동역학 분석 파이프라인과 `LLM` 기반 피드백 처리를 수행한다.
6. 분석 결과를 프론트엔드로 다시 전달한다.
7. 사용자는 시각화된 분석 결과와 피드백을 확인한다.

### MVP v1의 의미

이 MVP는 내가 최종적으로 구현하려는 `랙 거치형 다각도 영상 촬영 운동역학 분석 플랫폼`의 초기 검증 버전이다.

즉, 완성형 플랫폼 전체를 만드는 단계가 아니라 아래 핵심 가설을 검증하는 단계다.

- 운동 영상을 업로드 받아 분석 가능한 구조를 만들 수 있는가
- 인체 스켈레톤 추출과 운동역학 분석을 유의미하게 연결할 수 있는가
- 분석 결과를 사용자가 이해하기 쉬운 형태로 시각화할 수 있는가
- `LLM` 피드백이 사용자 경험에 실제 가치를 줄 수 있는가

### 랜딩페이지에서 전달해야 할 인상

- 기술적으로 설득력 있는 제품
- 복잡한 분석 과정을 직관적으로 보여주는 UI
- 실험적 프로젝트가 아니라 실제 제품으로 확장 가능한 초기 버전이라는 인상
- 운동 분석, 자세 피드백, 시각화가 자연스럽게 연결되는 경험

### UI 구성 방향

랜딩페이지는 아래 내용을 중심으로 구성하면 된다.

- 서비스 한 줄 소개
- 업로드부터 분석 결과 확인까지의 흐름
- `MediaPipe`, 운동역학 분석, `LLM` 피드백, 시각화의 연결 구조
- 이 MVP가 최종 플랫폼의 어떤 핵심 기능을 먼저 검증하는지에 대한 설명

### 필수 UI 요소

랜딩페이지 또는 메인 데모 영역에는 아래 요소가 반드시 포함되어야 한다.

- 영상 업로드 버튼
- `FPS` 선택 UI
- 분석 버튼
- 분석 결과 재생 버튼

이 네 가지는 단순 장식이 아니라 실제 제품 핵심 흐름을 보여주는 주요 인터랙션으로 표현되어야 한다.

### 시각화 및 재생 요구사항

분석 결과 화면에서는 아래 경험이 반드시 드러나야 한다.

- 분석된 인체 스켈레톤이 사용자 원본 영상 위에 그대로 오버레이되어 보여야 한다.
- 스켈레톤 시각화와 원본 영상은 시간축이 맞는 상태로 `sync` 재생되어야 한다.
- 사용자는 버튼을 통해 원본 영상과 분석 결과가 함께 재생되는 경험을 직관적으로 이해할 수 있어야 한다.

즉, 단순히 "분석이 끝났다"는 결과 화면이 아니라, 사용자의 실제 동작 위에 분석 스켈레톤이 겹쳐진 상태로 동기화 재생되는 제품 경험을 중심으로 UI를 구성해야 한다.

### 결과 데이터 표시 요구사항

분석 결과 영역에는 시각화만 있는 것이 아니라, 실제 분석 산출물도 함께 담겨 있어야 한다.

- 추출된 스켈레톤 `JSON` 데이터
- 운동역학 분석 결과 수치값
- `LLM`이 생성한 피드백 문장

이 정보들은 사용자가 "영상이 분석되었다"는 사실만 보는 것이 아니라, 어떤 데이터가 추출되었고 어떤 해석이 도출되었는지까지 확인할 수 있게 구성되어야 한다.

특히 결과 화면은 아래 세 요소가 모두 포함되어야 한다.

- 원본 영상 + 스켈레톤 오버레이 재생 영역
- 수치 기반 운동역학 분석 패널
- `JSON` 데이터 및 `LLM` 피드백 패널

이 세 요소는 반드시 한 화면에 동시에 밀집되어 있을 필요는 없다.

- 오버레이 영상 재생 영역은 메인 데모 섹션으로 강조해도 된다.
- 데이터 대시보드 영역은 별도 섹션으로 분리해도 된다.

중요한 것은 레이아웃의 분리 여부가 아니라, 사용자가 아래 흐름을 자연스럽게 이해할 수 있어야 한다는 점이다.

1. 영상을 업로드하고 분석을 실행한다.
2. 원본 영상 위에 스켈레톤이 오버레이된 결과를 동기화 재생으로 확인한다.
3. 별도의 데이터 섹션에서 `JSON`, 운동역학 수치값, `LLM` 피드백을 확인한다.

즉, 이 랜딩페이지의 데모는 단순한 마케팅용 목업이 아니라, 입력 데이터부터 분석 수치와 해석 결과까지 이어지는 전체 파이프라인이 섹션 단위로 명확하게 이해되도록 설계되어야 한다.
