### 5.1 좌표계 해석

이 데이터는 MediaPipe 계열 포즈 랜드마크와 유사한 정규화 좌표계로 보인다.

- `v`: 해당 랜드마크가 화면에서 얼마나 안정적으로 추정되었는지 나타내는 값

## 8. 백스쿼트 분석에서 바로 쓸 수 있는 파생 지표

이 파일 하나만으로도 다음 파생 지표를 만들 수 있다.

### 8.1 반복 구간 탐지

추천 기준:

- `hip` 또는 `shoulder`의 `y` 궤적을 평활화
- 국소 최소/최대점 탐지
- 하강, 최저점, 상승 구간 분리

실무적으로는 아래 조합이 안정적이다.

- 중심 고관절 `mid_hip_y = mean(left_hip_y, right_hip_y)`
- 중심 무릎 `mid_knee_y = mean(left_knee_y, right_knee_y)`
- 중심 어깨 `mid_shoulder_y = mean(left_shoulder_y, right_shoulder_y)`

### 8.2 관절 각도

대표 각도:

- 무릎 각도: `hip-knee-ankle`
- 고관절 각도: `shoulder-hip-knee`
- 발목 각도: `knee-ankle-foot_index`

각도 변화량으로:

- 최저점 깊이
- 상승 시 힙 드라이브 패턴
- 무릎 우선/엉덩이 우선 전략

을 볼 수 있다.

### 8.3 좌우 비대칭

비교 예시:

- `left_knee_y` vs `right_knee_y`
- `left_hip_z` vs `right_hip_z`
- `left_ankle_x` vs `right_ankle_x`

이상 징후:

- 한쪽 무릎의 과도한 내측 이동
- 한쪽 고관절의 하강 지연
- 발 간격 또는 체중 이동 불균형

### 8.4 몸통 기울기

다음 벡터를 사용한다.

- `mid_shoulder -> mid_hip`

이 벡터와 수직축/수평축의 각도를 계산하면:

- 전경사/후경사 변화
- 하강 시 torso lean
- 상승 시 hip shoot-up

등을 추적할 수 있다.

이 샘플은 전반적으로 품질이 양호하다.

- 프레임 수 누락 없음: `1074 / 1074`

다만 분석 시 아래 처리가 필요하다.

### 9.1 저신뢰 포인트 필터링

권장 규칙 예시:

- `visibility < 0.8`면 보수적으로 제외
- `0.8 <= visibility < 0.9`면 보간 후보
- `visibility >= 0.9`면 기본 사용

### 9.2 스무딩

포즈 데이터는 프레임 간 미세 진동이 존재하므로 아래 중 하나를 권장한다.

- 이동평균
- Savitzky-Golay 필터
- 저차 칼만 필터

특히 각도 계산 전 스무딩이 중요하다.

### 9.3 좌표 정규화 재정의

카메라 위치나 촬영 구도가 다르면 원시 `x`, `y`, `z`만으로 세션 비교가 왜곡될 수 있다.

권장 방법:

- 골반 중심을 원점으로 재정렬
- 어깨폭 또는 고관절 간 거리로 스케일 정규화
- 좌우 중점 기반 상대좌표 생성

## 10. 분석 시 주의할 점

이 데이터는 2D 화면 정규화 좌표 + 상대 깊이 추정치의 성격이 강하다. 따라서 다음 한계가 있다.
