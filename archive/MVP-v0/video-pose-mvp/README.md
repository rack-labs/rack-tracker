# Video Pose MVP

`index.html`의 `container video-section`과 `videoPose.js` 관련 모듈만 분리한 독립 실행용 정적 프로젝트입니다.

## 구성

- `index.html`: 비디오 스켈레톤 추출 UI
- `src/`: `videoPose.js` 및 관련 모듈
- `styles/`: 비디오 섹션에 필요한 최소 스타일
- `assets/video/backSquat.mp4`: 기본 샘플 비디오

## 실행

브라우저에서 직접 열어도 되지만, ES module/CORS 이슈를 피하려면 간단한 로컬 서버로 여는 편이 안전합니다.

## MediaPipe 사용 방식 평가

현재 구현은 "단일 인물의 리프팅 자세를 빠르게 추출하고, 스켈레톤 오버레이와 JSON 결과를 확인하는 MVP"라는 목표에는 대체로 적절하다.

- `PoseLandmarker`를 `VIDEO` 모드로 사용하고 `numPoses: 1`로 제한한 점은 단일 리프터 기준 MVP에 맞다.
- 브라우저에서 바로 실행되고 별도 백엔드 없이 JSON을 뽑을 수 있어 프로토타이핑 속도가 빠르다.
- 오버레이 재생과 skeleton export까지 한 번에 확인할 수 있어 후속 분석 실험용 입력 데이터를 만들기 쉽다.

다만 현재 방식은 정확도와 재현성보다 "빠르게 동작하는 데모" 쪽에 더 가깝다.

- 모델이 `pose_landmarker_lite`라서 속도는 유리하지만, 스쿼트 같은 동작에서 관절 위치 안정성은 `full` 또는 상위 모델보다 불리할 수 있다.
- `@latest` CDN과 원격 모델 URL에 의존하므로 결과 재현성과 장기 유지보수성이 약하다.
- 프레임마다 `seek` 후 `detectForVideo()`를 호출하는 구조라 긴 영상에서는 비효율적일 수 있고, 추적 기반 이점을 충분히 살리지 못한다.

## 현재 목표와의 적합성

아래 목표라면 현재 MediaPipe 사용 방식은 합리적이다.

- 단일 사람의 자세를 시각화한다.
- 백스쿼트 등 리프팅 영상에서 관절 좌표를 추출한다.
- 후처리용 skeleton JSON을 만든다.

아래 목표라면 현재 방식만으로는 최적화되어 있지 않다.

- 바벨 경로를 안정적으로 추적한다.
- 랙, 훅, 세이프티바와의 상대 위치를 판별한다.
- 언랙, 리랙, 반복 구간 같은 이벤트를 신뢰성 있게 인식한다.

이유는 현재 출력이 사람의 33개 포즈 랜드마크뿐이며, 바벨이나 랙 자체는 추적하지 않기 때문이다.

## 권장 방향

- 자세 추출 MVP를 유지할 경우:
  `PoseLandmarker` 기반 구조를 유지하되 모델 버전을 고정하고 `lite` 대신 `full` 성능을 비교한다.
- rack tracker 목표로 확장할 경우:
  사람 자세 추출은 MediaPipe Pose를 유지하고, 바벨/랙 추적은 별도 객체 검출 또는 키포인트 추적으로 분리하는 것이 적절하다.
- 분석 품질을 높일 경우:
  스무딩, 보간, confidence 기반 필터링을 후처리에 추가하는 편이 낫다.

## 결론

현재 MediaPipe 사용은 "비디오 자세 추출 MVP"에는 적절하지만, "rack tracker"의 핵심 목표까지 포함하면 최적화된 최종 방식은 아니다. 즉, 사람 자세 추출기로는 맞고, 바벨/랙 추적기까지 겸하려면 추가 추적 계층이 필요하다.
